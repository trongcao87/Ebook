01

Repository là gì?

Hiểu khái niệm trước khi bắt đầu

💡 Giải thích đơn giản

Hãy tưởng tượng Repository (gọi tắt là repo) giống như một thư mục dự án trên đám mây.


📁
Lưu trữ toàn bộ code
Tất cả file, folder của dự án đều nằm trong repo.
🕐
Lưu lịch sử thay đổi
Mỗi lần bạn sửa code, GitHub ghi nhớ thay đổi đó. Bạn có thể quay lại phiên bản cũ bất cứ lúc nào.
👥
Làm việc nhóm
Nhiều người cùng chỉnh sửa dự án mà không lo bị ghi đè.
🌍
Chia sẻ với thế giới
Bạn có thể chia sẻ link repo để người khác xem, dùng, hoặc đóng góp vào dự án.
💬
Ví dụ thực tế: Bạn viết một website bán hàng. Thay vì lưu trên máy tính cá nhân (dễ mất), bạn lưu lên GitHub. Đồng nghiệp có thể truy cập, chỉnh sửa cùng lúc, và mọi thay đổi đều được ghi lại.
02

Truy cập trang tạo Repo

Bước đầu tiên: mở trình duyệt và vào đúng địa chỉ

🌐 Cách truy cập

1
Mở trình duyệt (Chrome, Firefox, Edge…)
2
Đăng nhập tài khoản GitHub tại github.com
3
Truy cập trực tiếp địa chỉ: github.com/new
🔒 github.com/new

↑ Đây là trang bạn sẽ thấy khi vào github.com/new

⚠️
Lưu ý: Bạn cần đăng nhập trước. Nếu chưa có tài khoản, hãy đăng ký miễn phí tại github.com/signup
03

Phần General

Điền thông tin cơ bản: tên & mô tả dự án

👤 Ảnh chụp phần General

🔒 github.com/new

👤 Trường: Owner (Chủ sở hữu)

Owner *
Đây là tài khoản sở hữu repo. Thường là tên tài khoản cá nhân của bạn, hoặc tên tổ chức/nhóm nếu bạn là thành viên.
trongcao87
💡
Khi nào chọn Organization? Nếu bạn làm việc cho công ty hoặc nhóm trên GitHub, chọn tên tổ chức thay vì tên cá nhân. Repo sẽ thuộc về tổ chức đó.

Hành động: Hầu hết người dùng mới giữ nguyên mặc định (tên tài khoản cá nhân).

📝 Trường: Repository name (Tên Repo)

Repository name *
Tên duy nhất cho repo của bạn. Tên này sẽ xuất hiện trong URL và là cách mọi người nhận ra dự án của bạn.
my-first-project

Quy tắc đặt tên tốt:

  • ✅ Dùng chữ thường, số, và dấu gạch ngang -
  • ✅ Ngắn gọn, dễ nhớ: bai-tap-html, portfolio-website
  • ❌ Tránh dấu cách — dùng - thay thế
  • ❌ Tránh ký tự đặc biệt: @, #, !…
⚠️
Tên repo không được trùng với repo khác trong cùng tài khoản. GitHub sẽ báo lỗi ngay nếu bị trùng.

💬 Trường: Description (Mô tả)

Description
Mô tả ngắn về dự án của bạn. Xuất hiện ngay dưới tên repo trên GitHub. Tối đa 350 ký tự.
Website portfolio cá nhân bằng HTML/CSS
💡
Có bắt buộc không? Không — nhưng nên điền. Một mô tả tốt giúp người khác (và chính bạn sau này) hiểu ngay repo này dùng để làm gì.
04

Phần Configuration

Cài đặt quyền truy cập và các tính năng khởi tạo

⚙️ Ảnh chụp phần Configuration

🔒 github.com/new

👁️ Cài đặt: Choose Visibility (Quyền truy cập)

Bạn muốn ai được nhìn thấy repo này?

🌍 Public Phổ biến nhất
Ai cũng có thể xem code của bạn. Phù hợp để chia sẻ portfolio, open source, bài tập học tập.
🔒 Private
Chỉ bạn và người được mời mới thấy. Phù hợp cho dự án công ty, dự án cá nhân chưa muốn công khai.
💡
Gợi ý cho người mới: Chọn Public nếu đây là bài tập hoặc dự án học tập. Chọn Private nếu có thông tin nhạy cảm (mật khẩu, API key trong code).

📄 Cài đặt: Add README

Add README
README là file README.md — trang "bìa" của repo. Khi ai vào repo của bạn, họ sẽ đọc file này đầu tiên. Nó thường chứa: mô tả dự án, cách cài đặt, cách sử dụng.
✅
Nên bật ON. Đặc biệt với người mới — bật README giúp repo không bị trống rỗng và dễ bắt đầu hơn. Bạn có thể chỉnh sửa nội dung README sau.

🙈 Cài đặt: Add .gitignore

Add .gitignore
File .gitignore liệt kê những file/folder không muốn đẩy lên GitHub. Ví dụ: file node_modules/ (nặng, không cần thiết), file .env (chứa mật khẩu bí mật).

Cách chọn:

  • → Dự án Node.js/React: chọn Node
  • → Dự án Python: chọn Python
  • → Chưa biết: giữ No .gitignore và thêm sau cũng được
💡
Nếu bạn chỉ làm HTML/CSS đơn giản, bỏ qua cài đặt này cũng không sao.

⚖️ Cài đặt: Add license (Giấy phép)

Add license
Giấy phép quy định người khác được làm gì với code của bạn — có được dùng thương mại không, có cần ghi tên tác giả không...
Giấy phépÝ nghĩaKhi nào dùng
MITCho phép làm gì cũng được, chỉ cần giữ tên tác giảDự án open source, bài tập
Apache 2.0Tương tự MIT nhưng có bảo vệ bằng sáng chếDự án lớn hơn
No licenseMặc định — mọi quyền thuộc về tác giảDự án riêng tư, học tập
💡
Gợi ý cho người mới: Chọn MIT License nếu muốn mọi người dùng tự do, hoặc No license nếu chưa chắc.
05

Tạo Repository

Bước cuối: bấm nút xanh lá và hoàn thành!

🚀 Bấm nút "Create repository"

Sau khi điền đầy đủ thông tin, kéo xuống cuối trang và bấm nút:

Create repository

GitHub sẽ tạo repo và chuyển bạn đến trang repo mới — thường có địa chỉ dạng:

github.com/tên-tài-khoản/tên-repo
🎉
Xin chúc mừng! Bạn đã tạo xong repository đầu tiên trên GitHub!

📋 Checklist trước khi bấm tạo

  • ✅ Owner đúng tài khoản của bạn
  • ✅ Tên repo ngắn gọn, không dấu cách, không trùng
  • ✅ Description có điền (không bắt buộc nhưng nên có)
  • ✅ Visibility đã chọn Public hoặc Private phù hợp
  • ✅ README đã bật nếu muốn repo có nội dung ban đầu
  • ✅ **.gitignore & License đã chọn phù hợp (hoặc để mặc định)
06

Tóm tắt & Bước Tiếp Theo

Nhìn lại toàn bộ quy trình

📊 Bảng tóm tắt tất cả các trường

TrườngBắt buộc?Gợi ý cho người mới
Owner✅ CóGiữ nguyên tên tài khoản cá nhân
Repository name✅ CóTên ngắn, dùng dấu -, không dấu cách
Description❌ KhôngNên điền 1 dòng mô tả ngắn
Visibility✅ CóPublic nếu muốn chia sẻ, Private nếu cần bảo mật
Add README❌ KhôngNên bật ON cho người mới
Add .gitignore❌ KhôngChọn theo ngôn ngữ (Node, Python...) hoặc bỏ qua
Add license❌ KhôngMIT nếu muốn open source, không thì bỏ qua

🚀 Bước tiếp theo sau khi tạo xong

1
Clone repo về máy
Dùng lệnh: git clone https://github.com/username/repo-name.git
2
Thêm file vào repo
Tạo file, dùng git add . → git commit -m "message" → git push
3
Chỉnh sửa README
Mở file README.md và viết mô tả chi tiết hơn về dự án
4
Chia sẻ link
Copy link repo và gửi cho bạn bè, nhà tuyển dụng, hoặc thầy giáo!
